> 评测日期 2026-07-11 ｜ 评测对象:上海大歌剧院项目全量图纸(2309 张)生成的工程 3D 模型 **版本 v6**(2026-07-10 构建)

## 0. 一句话结论

**建模管线能对 2309 张全量竣工图跑通并产出 ready 模型(未崩溃、优雅降级、语义分区/构件/审校闭环齐全),但当前 v6 模型在三维视口中渲染为空、且存在致命的楼层错乱与识别缺口,尚不可用于展示或算量交付。** 根因集中在「楼层号误提取导致竖向标高错乱」「贴图资源浏览器不可达」「板/梁识别缺口」三处,均可定位修复。与效果图的自由曲面造型存在**方法论层面的根本差距**(确定性楼层堆叠 ≠ 建筑曲面外壳),需明确能力边界。

## 2. 关键问题清单(按严重度)

### 🔴 P0-1 楼层号误提取 → 标高错乱(致命,是渲染空白的主因)

**证据**:42 层标高从 **基础层 -411.6m** 到 **屋面 +441.0m**,竖向跨度 852m。楼层列表出现 `14层/19层/22层/24层/…/52层/72层/75层/…/98层/屋面(99)`——真实建筑无这些楼层。

**根因**:
- `services/floor_parser.py` 从图名/图号/文本提取楼层号时,把**轴号、图号、尺寸数字**(如 52、72、98)误判为楼层号,造出几十个"幻影楼层";
- "基础层"命中哨兵 `order=-98`,该哨兵**未被隔离出标高计算**,-98 × 4.2m ≈ **-411.6m**;
- 幻影层按 4.5m 层高线性堆叠 → 442m 高塔。

**后果**:楼层被散布在 -411~+441m 的荒谬包围盒中,相机无法框住有效几何 → 视口看似空白;算量/堆叠全部失真。

### 🔴 P0-2 贴图资源浏览器不可达(渲染空白的第二主因)

**证据**:`MINIO_ENDPOINT=minio:9000`,而 **`MINIO_PUBLIC_ENDPOINT` 为空**。预签名 URL 以内网主机名 `minio:9000` 下发,浏览器无法解析 → `ERR_NAME_NOT_RESOLVED`。`scene.lod.default_mode="mixed"`(依赖贴图),贴图全挂 → 视口空白。

### 🔴 P0-3 相机取景无自适应/无离群剔除

大模型 + P0-1 的荒谬竖向跨度下,默认相机无法自动框住有效几何;贴图不可用时也未回退到「构件」几何模式。

### 🟠 P1-1 板严重漏识别

42 层仅 **19 块板**,首层 0 板。板识别(楼层轮廓/轴网包络启发式)对本项目几乎失效,直接影响体量与算量。

### 🟠 P1-2 梁欠识别 / 截面缺省

梁 750 << 柱 5078,首层 0 梁。梁靠平行线对启发式,漏检多且截面走硬编码默认(缺剖面回填)。

### 🟠 P1-3 未分层率 32%(739 张)

大量图纸(尤其机电、详图、总图)未能归层,进入待人工识别队列。分层依赖图名/图号文本,竣工图命名不规范时失效。

### 🟠 P1-4 机电线型噪声

管线 22869 数量异常,疑将大量非管线线条(引线、标注、图框)误识别为管线,污染构件与问题计数。

### 🟡 P2-1 LOD 卡 200

缺失证据:`scale / registered_grid / dimensions / cross_view_match / stable_component_boundaries / geometry_consistent`。未达算量级 LOD300。

### 🟡 P2-2 坐标配准存疑

柱坐标水平范围 x[-22.8, 427.7]、y[-0.4, 311](约 450×311m)。可能是真实米制大尺度,也可能是多套图纸未配准到共同原点导致横向铺开;`cross_view_match` 缺失佐证配准不足。

## 3. 建模模块功能验证

### ✅ 可用/表现良好

- **全量鲁棒性**:2309 张图纸(4 套盖章竣工图)构建管线跑通,ready v6,未崩溃,单点失败优雅降级。
- **语义分区**:自动产出 14 分区(北/南/东/西/A~D 区,置信 58~84%)、61 功能空间(舞台等)、3 单体分组。
- **构件识别产出**:柱/墙数量级合理,YOLO 设备补充接线正常。
- **审校闭环**:待人工识别 739、待审语义 80 正确进队列;返工度量、语义树操作(确认/改类/合并等)UI 正常。
- **质量诚实**:LOD 如实停在 200 不虚高;未分层/低置信度单体如实计数。

### ❌ 不可用/失效

- **三维展示**:视口空白,不可用于展示交付(P0-1/2/3)。
- **楼层堆叠**:楼层数与标高严重失真(P0-1),堆叠结果不可信。
- **板/梁几何**:板近乎缺失、梁欠识别(P1-1/2),影响体量与算量。
- **算量级精度**:LOD 未达 300(P2-1),QTO 结果不可作结算依据。

## 4. 优化升级与改进方案(按优先级)

### P0 — 修复致命项(先让模型"能看、可信")

1. **楼层号提取加护栏**(`services/floor_parser.py`)
   - 楼层关键词白名单化(`地下N层/N层/首层/屋面/夹层/设备层/避难层`),**数值范围钳制**(地上 ≤ 合理层数、地下 ≤ 合理层数);
   - 剔除轴号(①-⑳/字母轴)、图号、尺寸数字等**非楼层数字**;
   - "基础层/桩基"等哨兵**不参与标高线性计算**,单独钳位;
   - 接入 `section_z_recovery` 剖面标高做**真值锚定**,替代 4.5m 线性假设。
2. **MinIO 公网端点**:配置 `MINIO_PUBLIC_ENDPOINT=localhost:9002`(或部署域名),预签名 URL 一律走 public endpoint,消除 `ERR_NAME_NOT_RESOLVED`。
3. **相机自适应 + 离群剔除**(前端 `ModelViewer`/`FragmentsScene`):按有效几何包围盒自动取景,剔除标高离群楼层;贴图不可用时**自动回退「构件」几何模式**,并对空场景给出明确提示而非静默空白。

### P1 — 提升识别质量

4. **板识别增强**:楼层封闭轮廓/轴网包络 + 结构板图层约定 + 剖面板厚回填。
5. **梁识别与截面**:平行线对 + 梁配筋图层线索;截面走 `model_component_sections` 剖面回填,替换硬编码。
6. **机电线型去噪**:按图层/线型过滤引线/标注/图框,管线只取给排水/暖通/电气系统图层;设备优先 YOLO + 图层双确认。
7. **降未分层率**:文件名解析规则扩充 + 图签区 OCR 补图号 + 批量人工归层工具(现有 DrawingAnnotationQueue 深化)。

### P2 — 能力升级

8. **冲 LOD300**:补比例尺检测、轴网配准(`grid_anchor_extractor`)、剖面 z(`cross_view_z`)三项证据,点亮 `cross_view_match`。
9. **多套图配准**:围护/建筑/结构/机电四套图以轴网锚点配准到共同原点,消除横向铺开。
10. **自由曲面造型(边界与增强)**:
    - 明确对外话术:交付物是**审图/算量骨架**,非建筑曲面效果图;
    - 如需展示级外壳,单列**旁路增强**(网格重建/参数化屋面),仅作可视化,**不进算量**,与主线解耦。

---

## 5. 结论

建模模块的**管线韧性、语义与审校闭环**已具备工程可用基础,但 v6 在上海大歌剧院这一真实旗舰项目上暴露出**楼层错乱、渲染空白、板梁缺口**三类致命/高危问题,当前**不可用于展示与算量交付**。上述 P0 三项修复后可恢复"能看、堆叠可信";P1 补齐识别质量后可支撑体量与初步算量;P2 决定是否迈向 LOD300 与展示级外壳。与效果图的自由曲面差距是路线边界,应明确标注、避免过度承诺。

> 复现要点:模型 `project_id=9188e163-c684-415e-a4ec-08f208273eff`,`GET /projects/{id}/model` 读 scene;标高错乱见 `scene->'floors'[*]->'elevation_m'`;渲染空白见浏览器控制台 `ERR_NAME_NOT_RESOLVED` 与 `MINIO_PUBLIC_ENDPOINT` 为空。
