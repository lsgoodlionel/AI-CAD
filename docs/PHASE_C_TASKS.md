## 0. 阅读前必须建立的三条心智

1. **能力边界（诚实写在最前，用于向上管理预期）**
   - 全自动出**生产级 BIM 今天不存在**。学术峰值实例精度 ~94%，但 BIM 要求拓扑闭合（墙首尾相接）、语义正确、可结算，单点 6% 错误率在整栋楼累积成大量返工。
   - Phase C 的现实目标是**符号识别学习模型补规则的模糊边界**（不替代规则）+ **人工审校工作台把返工点收敛**。
   - 全流程「AI 出初模 → 人工审改」的**效率提升现实值 25–30%**，任何 KPI/对外话术都以此为锚，不承诺「自动出图」。

2. **关键路径 = 数据冷启动（最大隐形成本）**
   - 所有开源 SOTA（CADTransformer/VecFormer/SymPoint）**只覆盖建筑平面图（FloorPlanCAD）**。
   - **结构（柱梁板）/ 机电管线 / 装修 + 中文施工图域几乎零现成权重，必须自建数据集。** 低估这点会让「上学习模型」这步无限延期。这是 Phase C 的**关键路径与最大隐形成本**。

3. **许可证陷阱（写代码前必须过法务）**
   - 可商用交集只有：`CADTransformer(MIT ✅)`、`VecFormer(Apache 2.0 ✅，权重暂未释放)`。
   - `SymPoint V1/V2`（非商用 ⛔ / 未声明 ⛔）**只能作内部评测对标天花板，绝不进产品代码**。
   - `ArchCAD-400K` 数据集许可待确认——**只借鉴其「用 CAD 图层/块属性自动标注」的方法论**，不直接分发其数据前先核许可。

### C-01　开源件许可证合规审计（独立卡点任务）

- **描述**：在任何模型代码落地**之前**，逐一核验 Phase C 拟用开源件的许可证与可商用性，输出书面结论并作为后续任务的准入门禁。覆盖：CADTransformer、VecFormer、SymPoint、ArchCAD-400K、其依赖树（torch/dgl/torch-geometric 等）以及数据集来源。
- **交付物**：`docs/PHASE_C_LICENSE_AUDIT.md`（逐项：库名 / 版本 / 许可证 / 可商用 ✅⛔ / 权重许可 / 依赖传染性 / 结论）；CI 中新增 license 检查清单条目。
- **涉及·新增文件**：`docs/PHASE_C_LICENSE_AUDIT.md`（新增）、`.github/workflows/ci.yml`（增补 license 检查 step）。
- **依赖**：无（**最前置，所有 G3/G4 任务的硬门禁**）。
- **开源件**：CADTransformer **MIT ✅**；VecFormer **Apache 2.0 ✅**（权重未释放）；SymPoint V1 **非商用 ⛔** / V2 **未声明 ⛔**；ArchCAD-400K 数据集**许可待确认（仅借方法论）**。
- **工作量**：S（2 人天）。
- **验收标准**：每个拟引入件均有明确 ✅/⛔ 结论；⛔ 件在代码库有隔离约束（不进 `requirements.txt`、不进生产镜像）；法务/负责人签字。
- **风险**：SymPoint 依赖易被间接引入（如借其数据加载器）→ 必须物理隔离到独立评测仓/容器（见 C-11）。

### C-02　DXF→SVG/图元 JSON 预处理器

- **描述**：构建统一预处理器，把 DXF/DWG（经现有 `dwg_support` 转 DXF）转换为**所有开源模型能吃的 SVG + 图元 JSON**（模型不吃 DXF/DWG，这是硬约束）。输出图元级结构：primitive 类型、坐标序列、图层、块引用、颜色/线型。为 C-04 自动标注与 C-08/C-09 模型输入提供统一入口。
- **交付物**：预处理器模块 + CLI + 单元测试；输出 schema 文档。
- **涉及·新增文件**：`apps/api/core/model3d/preprocess/__init__.py`、`dxf_to_svg.py`、`primitive_json.py`、`schema.py`（新增）；复用 `apps/api/core/model3d/geometry_extractor.py`（L1 已提精确图元，尽量复用其提取，避免重复造轮子）；`apps/api/core/ai_review/dwg_support.py`（DWG→DXF）。
- **依赖**：C-01（许可前置）；软依赖 Phase A 的 ezdxf 栈已在镜像内。
- **开源件**：`ezdxf`（MIT ✅）；SVG 序列化用 `svgwrite`(MIT ✅) 或手写。
- **工作量**：L（12 人天）。
- **验收标准**：给定 10 张真实矢量图纸，100% 产出结构合法的 SVG + JSON；坐标与 `geometry_extractor` 提取的图元对齐（抽样比对误差 < 1e-3）；异常图纸优雅降级不抛异常（对齐现有 `element_recognizer` 风格）。
- **风险**：DXF 方言/自定义实体多；曲线/样条离散化精度与模型输入约定需锁定；A0/A1 大幅面需切图策略（对齐第三章「大幅面细节进模型前丢失」的告警）。

---

### C-03　图层/块 INSERT 展开 + 坐标归一化子模块

- **描述**：C-02 的核心子能力单独拆分：递归展开 block `INSERT`（含嵌套块、缩放/旋转/阵列变换），把图元摊平到模型空间；再做坐标归一化（平移到原点 + 尺度归一，供神经网络输入）。同时**保留块名/图层作为弱标签溯源字段**（供 C-04 复用）。
- **交付物**：展开器 + 归一化器 + 单测（含嵌套块、MINSERT 阵列用例）。
- **涉及·新增文件**：`apps/api/core/model3d/preprocess/block_expander.py`、`normalize.py`（新增）。
- **依赖**：C-02。
- **开源件**：`ezdxf`（MIT ✅）。
- **工作量**：M（5 人天）。
- **验收标准**：嵌套块/阵列展开后图元数量与手工核对一致；归一化后坐标落在约定域（如 [0,1] 或单位尺度）；块名/图层元数据无丢失。
- **风险**：MINSERT 阵列、外部参照 XREF、匿名块的边界情况；深层嵌套性能。

---

### C-04　自动标注引擎（CAD 图层/块属性 → 弱标签）【关键路径】

- **描述**：借鉴 `ArchCAD-400K`「用 CAD 内在图层/块属性自动标注」思路，构建规则驱动的弱标注引擎：中国施工图图层高度规范化（A-WALL/S-COLU/M-*/E-*）+ 块名约定 → 自动生成符号类别弱标签。这是数据冷启动的**发动机**，把无标注图纸批量转成带弱标签样本，人工只需在 C-06 精修。
- **交付物**：自动标注引擎 + 图层/块名→类别映射表（YAML，可维护）+ 弱标注质量报告。
- **涉及·新增文件**：`apps/api/core/model3d/dataset/auto_label.py`（新增）、`apps/api/data/model3d/layer_class_map.yaml`（新增，参照现有 `data/rules/*.yaml` 风格）；复用 C-03 保留的图层/块元数据。
- **依赖**：C-02、C-03。
- **开源件**：**方法论借鉴 ArchCAD-400K**（数据集许可待确认，**不直接分发其数据**，仅复刻自动标注思路）；`ezdxf`（MIT ✅）。
- **工作量**：L（12 人天）。
- **验收标准**：对 ≥50 张真实图纸自动产出弱标签，弱标签与人工抽检一致率 ≥ 70%（弱标签本就有噪声，目标是冷启动而非终检）；映射表覆盖结构/机电/装修主要图层约定。
- **风险**：图层命名不规范/项目自定义图层 → 召回下降；不同设计院图层习惯差异大，映射表需持续养护。**这是关键路径任务，排期务必留足并前置启动。**

---

### C-05　中文专业域数据集冷启动（结构/机电/装修）【关键路径】

- **描述**：以 C-04 弱标签为种子，系统性构建**覆盖结构/机电/装修 + 中文域**的自建数据集（开源 SOTA 只覆盖建筑平面图，这是必做、无捷径）。定义类别体系（symbol taxonomy）、采集图纸样本、组织弱标注→人工精标注流转（对接 C-06）。分专业分批交付，先结构、再机电、后装修。
- **交付物**：`docs/PHASE_C_DATASET_SPEC.md`（类别体系 + 采集规范 + 各专业目标样本量）；首批可训练数据集（结构域，≥ 目标规模）。
- **涉及·新增文件**：`docs/PHASE_C_DATASET_SPEC.md`（新增）、`apps/api/data/model3d/dataset/`（数据集目录约定，**大文件走 MinIO/DVC，不进 git**）。
- **依赖**：C-04。
- **开源件**：**方法论借鉴 ArchCAD-400K**；评测对标类别对齐 FloorPlanCAD（供 C-14 交叉参照）。
- **工作量**：L（12 人天，且为**滚动持续投入**，非一次性）。
- **验收标准**：类别体系评审通过；结构域首批样本量达到可微调门槛（参考 CADTransformer 训练量级）；样本覆盖真实项目多样性（≥N 个项目、多设计院）。
- **风险**：**最大隐形成本**。图纸涉密→采集/脱敏合规；类别体系一旦返工代价极高，需一次评审到位；机电管线/装修域标注专业性强、成本高。**关键路径，最易被低估。**

### C-06　人工精标注管线 + 标注规范 + 质检

- **描述**：搭建把 C-04 弱标签精修为金标签的人工标注流程：标注规范文档、标注工具选型（优先复用/扩展前端审校工作台 C-16，避免另起炉灶）、双人交叉 + 仲裁质检、标注一致性（IoU/Kappa）度量。
- **交付物**：`docs/PHASE_C_ANNOTATION_GUIDE.md`；标注质检报告模板；金标签数据回流入 C-07 数据集版本。
- **涉及·新增文件**：`docs/PHASE_C_ANNOTATION_GUIDE.md`（新增）；标注工具复用 `apps/web/src/pages/model/ProjectModel/DrawingAnnotationQueue.tsx`（C-16 深化）。
- **依赖**：C-04、C-05；工具侧依赖 C-16。
- **开源件**：标注工具若外采需过 C-01 许可（如 label-studio 为 Apache 2.0 ✅，可选）。
- **工作量**：L（12 人天，含持续标注运营）。
- **验收标准**：标注规范通过评审；双标一致性 Kappa ≥ 0.8；每批金标签有质检记录可追溯。
- **风险**：标注人力是持续成本；专业构件（机电）需懂图人员，非通用众包可替代。

- **描述**：为自建数据集建立版本化管理（DVC 或 MinIO + 清单）、可复现的 train/val/test 切分（按项目而非按图切，防泄漏）、数据卡（datasheet）。保证 C-09 微调与 C-14 评测在**固定、可复现**的切分上进行。
- **交付物**：数据集版本清单 + 切分脚本 + 数据卡。
- **涉及·新增文件**：`apps/api/scripts/model3d/dataset_split.py`（新增）、`apps/api/data/model3d/dataset/DATASHEET.md`（新增）。
- **依赖**：C-05、C-06。
- **开源件**：`DVC`（Apache 2.0 ✅，可选）。
- **工作量**：M（5 人天）。
- **验收标准**：切分可一键复现（固定随机种子）；无同项目跨切分泄漏；test 集冻结，仅 C-18 最终评测解锁一次。
- **风险**：按图切分会导致同项目样本泄漏、评测虚高——必须按项目切分。

### C-08　CADTransformer 跑通全链路（PoC 基线）

- **描述**：用官方 CADTransformer(MIT) + 官方权重，跑通 `DXF→(C-02)SVG/JSON→spotting` 全链路，在 FloorPlanCAD 上复现论文量级指标（PQ≈68.5），确立**可运行基线**。此阶段只验证管线打通，不动自建数据。
- **交付物**：可运行推理脚本 + 复现指标报告 + 环境/依赖锁定（Dockerfile 片段）。
- **涉及·新增文件**：`apps/api/core/model3d/spotting/cadtransformer/`（推理封装，新增）、`apps/api/core/model3d/spotting/README.md`（新增）；输入对接 C-02 预处理器。
- **依赖**：C-01（许可 ✅）、C-02。
- **开源件**：**CADTransformer（MIT ✅，代码+权重齐全，开箱度 3/5）**。
- **工作量**：M（5 人天）。
- **验收标准**：在 FloorPlanCAD 子集复现 PQ 在论文 ±3 内；用 C-02 预处理的自有图纸能跑出（未微调可接受低分）符号候选输出。
- **风险**：CADTransformer 依赖较老（dgl/torch 版本）→ 环境地狱，需锁版本；官方权重仅建筑平面图，自有图纸零样本表现会很差（预期内，C-09 解决）。

### C-09　CADTransformer 在自建数据集微调

- **描述**：在 C-07 的自建中文专业域数据集上微调 CADTransformer，产出覆盖结构/机电/装修域的领域模型。分专业迭代（先结构）。产出可服务化的权重 + 训练配置。
- **交付物**：领域微调权重（存 MinIO）+ 训练配置 + 训练/验证曲线 + 分专业指标。
- **涉及·新增文件**：`apps/api/core/model3d/spotting/cadtransformer/train.py`、`configs/`（新增）；权重不进 git。
- **依赖**：C-07、C-08。
- **开源件**：**CADTransformer（MIT ✅）**。
- **工作量**：L（12 人天，含多轮调参）。
- **验收标准**：在自建 test 集上，结构域 PQ **显著高于**零样本基线（C-08）与纯规则基线（C-14 对比）；机电/装修域随数据到位滚动达标。里程碑 M2 的核心产出。
- **风险**：数据量不足→过拟合；GPU 资源与训练周期；类别不均衡（装修符号长尾）。

- **交付物**：`docs/PHASE_C_VECFORMER_WATCH.md`（跟踪记录 + 迁移评估 + 适配层接口草案）。
- **涉及·新增文件**：`docs/PHASE_C_VECFORMER_WATCH.md`（新增）；预留 `spotting/vecformer/`（占位）。
- **依赖**：C-01、C-02。
- **开源件**：**VecFormer（Apache 2.0 ✅，权重待释放，开箱度 2/5）**。
- **工作量**：S（2 人天，周期性复查）。

- **风险**：权重可能长期不释放→迁移无限期挂起；不阻塞主线（CADTransformer 已可用）。

### C-11　SymPoint 天花板对标（隔离环境，仅评测）

- **描述**：在**物理隔离的评测环境**（独立仓/容器，不进生产镜像、不进 requirements）用 SymPoint 系（非商用 ⛔）跑自建 test 集，作为**内部精度天花板参照**，量化「可商用模型（CADTransformer/VecFormer）距 SOTA 有多远」。**代码与权重绝不进产品，结论只以数字形式回流评测报告。**
- **交付物**：隔离评测记录 + 天花板对标数字（进 C-14 报告）。
- **涉及·新增文件**：**独立评测仓 `research/sympoint-eval/`（不纳入产品构建，`.dockerignore`/`.gitignore` 排除或单独仓库）**。

## 4. 里程碑与 Phase C 验收总标准

### 4.1 里程碑

| 里程碑 | 名称 | 达成判据 | 依赖任务 |
|---|---|---|---|
| **M0** | 合规与管线打通 | C-01 许可结论签字；C-02 预处理器可产 SVG/JSON；C-08 CADTransformer 在 FloorPlanCAD 复现基线 | C-01、C-02、C-08 |
| **M1** | **符号识别在专业域超纯规则** | 在自建 test 集上，学习模型（C-09）或融合（C-13）在**至少结构域**的 PQ/召回**显著优于纯规则**（C-14 量化，SymPoint 为天花板参照） | C-09、C-13、C-14 |
| **M2** | **审校工作台收敛返工点** | 深化后的 SemanticReviewQueue/DrawingAnnotationQueue 上线；C-17 看板显示返工率随迭代下降；人审修正可回流为训练标签 | C-15、C-16、C-17 |
| **M3** | 验收收口 | test 集终评达标；能力边界与遗留风险如实成文 | C-18 |

### 4.2 Phase C 验收总标准（全部满足方可验收）

1. **合规**：产品代码库不含任何非商用件（SymPoint）依赖；CADTransformer(MIT)/VecFormer(Apache) 许可结论书面签字；C-01/C-11 隔离边界经核验。
2. **模型精度**：在**自建专业域 test 集**上，符号识别学习模型/融合在**至少结构域超过纯规则基线**（对齐第四章 Phase C 验收「符号识别在你的专业域数据上超过纯规则」）；机电/装修域给出明确达标路线与当前差距（诚实标注未达标项）。
3. **融合正确性**：融合遵循「学习模型补规则模糊边界，不替代规则」——规则强命中不被覆盖，输出均带 `source` + `confidence`。
4. **审校闭环**：3D 重建结果作为待审成果接入三审文化；人审拓扑闭合/命名/规范符合性；返工点经 C-17 度量呈**收敛趋势**（对齐 Phase C 验收「审校工作台把人工返工点收敛」）。
5. **数据资产**：自建数据集版本化、按项目切分无泄漏、有数据卡与标注规范；人审修正形成**数据飞轮**回流。
6. **能力边界如实**：验收报告明确写明「全自动生产级 BIM 不存在，定位 AI 出初模 + 人工审改，效率提升现实值 25–30%」，不承诺一键出 BIM。
