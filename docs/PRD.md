## 一、背景与目标

### 问题定义

建筑施工企业在图纸深化环节存在三大脱节：

| 脱节类型 | 表现 | 损失 |
|---------|------|------|
| 技术与商务脱节 | 深化设计师只懂规范，不懂价格 | 无法量化优化收益 |
| 设计与结算脱节 | 图纸表达不清，结算被审减 | 已做工作无法收回价款 |
| 过程与劳务脱节 | 劳务凭经验施工，损耗超规范3倍 | 钢筋、砌体等材料浪费 |

**量化损失**（以年产值 100 亿为例）：

| 领域 | 精细化后可节约 |
|------|--------------|
| 钢筋翻样 | 约 1.25 亿元 |
| 管线综合 | 约 5000 万元 |
| 砌体/装修排版 | 约 3000 万元 |
| 节点优化 | 约 8000 万元 |
| **合计** | **近 3 亿元** |

### 产品目标

构建一套数字化图纸深化全过程管理平台，实现：

1. **流程管控**: 强制执行"三审三算"审批闭环，消除人为绕过 ✅（核心约束已实现；重大变更集团多人审批路径待完善）
2. **经济核算**: 每张深化图自动关联经济测算，量化优化价值 ✅（钢筋翻样 GB50010-2010 + FFD+2-opt 下料优化已实现）
3. **创效激励**: 在线管理创效提案、核算净节约额、执行分配兑现 ✅（已完整实现：测算/签字/铁三角分配/凭证 PDF）
4. **AI 审图**: 将规范审查时间从 3-5 天压缩至 30 分钟以内 🔶（四引擎 + LangGraph + YOLOv8 已实现；强条检出率目标指标待验证）
5. **知识积累**: 沉淀企业专属标准图集与错漏碰缺案例库 🔶（规范三途径导入 + NLP 流水线已实现；标准图集管理 / 案例库待实现）
6. **模型管理**: 所有 AI 引擎的模型和参数均可在后台实时配置，无需重启 ✅（已完整实现）

### 3.2 AI 智能审图模块（F-AI）🔶 框架已完整实现，验收指标待验证

```
图纸文件
    ↓ VisionEngine（ezdxf / PyMuPDF / PaddleOCR 降级 / YOLOv8 图元检测）✅
    ↓
    ├── 引擎 1：RulesEngine（YAML DSL，条件评估器）✅
    │    └── 5 个专业（common/structure/architecture/mep/decoration）✅
    │
    ├── 引擎 2：KGEngine（AGE Cypher + SQL 降级）✅
    │    └── 图谱查询已实现；规范 NLP 提取流水线已实现 ✅
    │
    ├── 引擎 3：RAGEngine（Chroma + LangGraph 三步推理）✅
    │    └── LangGraph Agent：identify → lookup → synthesize，graceful degradation ✅
    │
    └── 引擎 4：VisionEngine（ezdxf + fitz + PaddleOCR + YOLOv8）✅
         └── 文字提取 ✅ + 图元检测（yolo_detector.py，graceful degradation）✅
    ↓
    结果合并（按 severity 排序写入 ai_review_issues）✅
    → 批注版 PDF + 清单版 Excel 报告生成 ✅（ai_report_generator.py）
```
