## 一、背景与目标

### 问题定义

建筑施工企业在图纸深化环节存在三大脱节：

| 脱节类型 | 表现 | 损失 |
|---------|------|------|
| 技术与商务脱节 | 深化设计师只懂规范，不懂价格 | 无法量化优化收益 |
| 设计与结算脱节 | 图纸表达不清，结算被审减 | 已做工作无法收回价款 |
| 过程与劳务脱节 | 劳务凭经验施工，损耗超规范3倍 | 钢筋、砌体等材料浪费 |

**量化损失**（以年产值 100 亿为例）：

| 领域 | 精细化后可节约 |
|------|--------------|
| 钢筋翻样 | 约 1.25 亿元 |
| 管线综合 | 约 5000 万元 |
| 砌体/装修排版 | 约 3000 万元 |
| 节点优化 | 约 8000 万元 |
| **合计** | **近 3 亿元** |

### 产品目标

构建一套数字化图纸深化全过程管理平台，实现：

1. **流程管控**: 强制执行"三审三算"审批闭环，消除人为绕过 🔶（核心约束已实现，部分边缘路径待完善）
2. **经济核算**: 每张深化图自动关联经济测算，量化优化价值 ❌（经济测算引擎待实现）
3. **创效激励**: 在线管理创效提案、核算净节约额、执行分配兑现 ✅（已实现；凭证 PDF 待实现）
4. **AI 审图**: 将规范审查时间从 3-5 天压缩至 30 分钟以内 🔶（四引擎框架已实现；报告生成待实现）
5. **知识积累**: 沉淀企业专属标准图集与错漏碰缺案例库 ❌（规范知识库管理待实现）
6. **模型管理**: 所有 AI 引擎的模型和参数均可在后台实时配置，无需重启 ✅（已完整实现）

**实现**: `POST /api/v1/technical-reviews` — transitions 状态机守卫

**强制动作**:

#### F-WORKFLOW-003: 二审（经济最优化）【一票否决核心节点】✅

**系统约束**:

**输出**: 《经济最优深化图》（含经济师数字签名）🔶（签字记录已存库，PDF 凭证待生成）

#### F-WORKFLOW-004: 三审（结算合规化）✅

**实现**: `POST /api/v1/settlement-reviews` — 403 `QUOTA_SHEET_MISSING` 错误码

**强制动作**:

- 确认节点大样与清单描述一致性 ✅
- 生成《限额领料单》✅（quota_sheet MinIO 文件上传）

**系统约束**:
- 未同步生成《限额领料单》，图纸无法发布到班组视图 ✅（API 层 403）

**输出**: 《可结算深化蓝图》（正式版，带三审签字）🔶（审批记录已存，PDF 蓝图待生成）
