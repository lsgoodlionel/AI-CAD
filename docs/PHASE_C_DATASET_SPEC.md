## 0. 为什么必须自建（一句话背景）

所有开源 SOTA（CADTransformer / VecFormer / SymPoint）**只在建筑平面图数据集 FloorPlanCAD 刷分**。
结构（柱梁板）/ 机电管线 / 装修 + **中文施工图域几乎零现成权重**——这是 Phase C 的**关键路径与最大隐形成本**，
无捷径。本文档定义「造什么数据、怎么造、造多少、怎么切」，是 C-04 / C-06 / C-07 / C-14 共同锚定的单一事实源。

## 1. 类别体系（Symbol Taxonomy）

### 1.1 顶层锚点（强制，不得另起炉灶）

顶层构件类别**固定 9 类**，与确定性识别栈完全对齐，来源：
`apps/api/core/model3d/layer_conventions.py::classify_by_layer` 与 `apps/api/data/layer_conventions.yaml`。

| # | 顶层类 | 中文名 | 说明 |
|---|--------|--------|------|
| 1 | `column` | 柱 | 结构竖向承重构件 |
| 2 | `beam` | 梁 | 结构水平承重构件 |
| 3 | `slab` | 板 | 楼板 / 现浇板 |
| 4 | `wall` | 墙 | 剪力墙 / 建筑墙 / 砌体 |
| 5 | `door` | 门 | 门（洞口） |
| 6 | `window` | 窗 | 窗（洞口） |
| 7 | `pipe` | 管线 | 机电管线（风/水/桥架） |
| 8 | `equipment` | 设备 | 机电设备 |
| 9 | `axis` | 轴网 | 定位轴线 / 轴网 |

机电系统维度**固定 4 类**（与 `layer_conventions.py::classify_system` 一致，仅对 `pipe` / `equipment` 有意义）：

> 类别体系一旦返工，已标注数据全部作废，代价极高（见 §5 诚实边界）。

### 1.3 与 FloorPlanCAD 类别的交叉参照（供 C-14 评测对齐）

FloorPlanCAD（35 类，以建筑平面图为主）是唯一有公开权重的对标基准。C-14 评测需在**可交叉的类别**上
把自建模型与开源基线对齐比较。下表给出映射关系，**不可交叉的专业域类别（结构/机电/装修主体）明确标注
"FloorPlanCAD 无对应"**——这正是自建数据集的价值所在。

| 本体系子类 | 顶层类 | FloorPlanCAD 对应类 | 对齐说明 |
|---|---|---|---|
| `door_single` | door | `single door` | 直接对齐 |
| `door_double` | door | `double door` | 直接对齐 |
| `door_sliding` | door | `sliding door` | 直接对齐 |
| `door_fire` | door | （门大类近似） | 防火门为中文域细分，FloorPlanCAD 不区分，评测时合并到 door |
| `window_normal` | window | `window` | 直接对齐 |
| `window_bay` | window | `bay window` | 直接对齐 |
| `window_blind` | window | `blind window` | 直接对齐 |
| `wall_shear` / `wall_infill` | wall | `wall` / `curtain wall` | 顶层对齐；剪力墙/砌体为结构语义细分，FloorPlanCAD 不含 |
| `axis_grid` | axis | 无对应 | FloorPlanCAD 无轴网类，仅本体系有 |
| `column_*`（全部） | column | **无对应** | **结构柱：FloorPlanCAD 完全不覆盖——自建必要性核心证据** |
| `beam_*`（全部） | beam | **无对应** | **结构梁：同上** |
| `slab_*`（全部） | slab | **无对应** | **楼板：同上** |
| `pipe_*`（全部） | pipe | **无对应** | **机电管线：同上** |
| `equip_*`（全部） | equipment | **无对应** | **机电设备：同上** |
| `deco_*`（全部） | 各归属类 | **无对应** | **装修：同上** |

> **评测口径**：C-14 在 door/window/wall 子集上做「自建模型 vs CADTransformer/VecFormer 官方权重」交叉对比；
> 在 column/beam/slab/pipe/equipment 上开源基线为零样本（预期极差），凸显领域微调价值。

## 2. 采集规范

### 2.1 图纸来源

- **优先内部真实项目图纸**：来自平台已归档的深化图纸（三审三算流程沉淀），天然贴合业务分布。
- **补充多设计院外部图纸**：为覆盖图层/块命名习惯差异（见 §2.4 多样性），需引入非本方设计院图纸。
- **禁止**：网络爬取的来源不明图纸（许可 + 涉密双重风险）；AI 生成的合成图纸（除 C-18 Demo 用合成整套图外，不进训练集）。

### 2.2 矢量优先（硬约束）

- **只收矢量图纸**：DWG（经 `apps/api/core/ai_review/dwg_support.py` 转 DXF）+ **矢量 PDF**。
- **拒收扫描件 / 位图 PDF**：符号识别模型只吃 SVG + 图元序列（见 `docs/PHASE_C_PREPROCESS_SCHEMA.md`），
  扫描件无图层/块元数据 → C-04 自动标注无从下手，且 OCR/位图路线属另一条链（PaddleOCR），不在本数据集范围。
- **图层/块完整性检查**：入库前抽检 DXF 是否保留规范图层命名（S-/A-/M-/P-/E- 前缀或中文约定）；
  图层被炸平（explode 为单一图层）的图纸自动标注召回极低，降级为"仅人工标注候选"或退回。

### 2.3 涉密 → 脱敏合规流程（强制）

图纸普遍涉密，采集入库**必须先脱敏**，流程：

1. **授权确认**：确认该项目图纸用于内部模型训练已获授权，登记来源项目与授权范围。
2. **元数据脱敏**：清除标题栏中的项目名称、单位、地址、设计人签名、图签二维码等敏感字段
   （仅保留构件几何 + 图层/块 + 必要标注文本，如轴号/构件编号/标高）。
3. **文件名脱敏**：入库统一改为脱敏编号命名（见 `dataset/README.md` 命名规范），不含真实项目名。
4. **存储合规**：原始图纸与派生数据一律存 MinIO（AES-256，对齐平台既有存储约定），**不进 git**（见 §6）。
5. **可追溯登记**：脱敏映射（脱敏编号 ↔ 真实项目）单独加密登记，仅授权人可查，用于按项目切分（§4）与问题回溯。

### 2.4 多样性要求（防偏置）

自建数据集若全来自单一设计院/单一项目，模型会过拟合该院图层习惯，泛化崩塌。硬性要求：

- **多项目**：每个专业批次 **≥ 8 个不同项目**（结构首批下限，机电/装修可随采集进度滚动放宽起步但需登记缺口）。
- **多设计院**：覆盖 **≥ 3 家设计院**的图层/块命名习惯，验证 C-04 映射表鲁棒性。
- **多建筑类型**：住宅 / 公建 / 商业至少各有覆盖，避免单一结构体系（如全剪力墙住宅）偏置。
- **图层习惯登记**：每个来源项目登记其图层命名体系（AIA 式 / 国标中文式 / 院内私有），供 C-04 映射表养护。

## 3. 各专业目标样本量

**滚动分批交付**：先结构 → 再机电 → 后装修（与 C-09 微调迭代顺序一致，先结构域达标）。
"样本"以**单张图纸内的标注符号实例数**为主口径，同时登记**图纸张数 / 项目数**。

| 批次 | 专业 | 冷启动可微调门槛（实例量级） | 图纸张数（下限） | 项目数（下限） | 交付节奏 |
|---|---|---|---|---|---|
| 第 1 批 | 结构（柱/梁/板/墙 + 建筑门窗/轴网） | **≥ 3–5 万实例** | ≥ 150 张 | ≥ 8 | C-05 首批交付，卡 C-09 结构域微调 |
| 第 2 批 | 机电（管线 + 设备，4 系统） | ≥ 2–3 万实例 | ≥ 120 张 | ≥ 6 | 结构达标后滚动启动 |
| 第 3 批 | 装修（长尾） | ≥ 1 万实例（长尾类可更少，靠规则兜底） | ≥ 80 张 | ≥ 5 | 最后，成本最高，优先规则+少量精标 |

**门槛量级依据**：对齐 CADTransformer 训练量级（FloorPlanCAD 约 1.5 万张平面图、百万级符号实例）。
自建数据**做领域微调而非从零预训练**，故所需量显著低于全量预训练——目标是"够微调出显著优于零样本+纯规则基线"
（C-09 / C-14 验收判据），而非追平 FloorPlanCAD 规模。样本量随 C-14 评测曲线**边采边评**动态调整：
若结构域某子类 PQ 未随数据增长提升，优先补该子类样本而非盲目扩量。

> 数据来源以 C-04 弱标签为**种子批量转换**，C-06 人工精修为**金标签**。弱标签负责规模冷启动（一致率目标 ≥70%，
> 见 C-04），金标签负责质量（Kappa ≥ 0.8，见 C-06）。训练用金标签 + 高置信弱标签混合，比例随 C-14 评测定。

## 4. 切分口径（train / val / test）

**唯一正确口径：按项目切分（split by project），而非按图切分（split by drawing）。**

- **原因**：同一项目的多张图纸共享图层习惯、构件编号体系、绘图风格。按图切分会让同项目样本同时落入
  train 和 test → 测试集信息泄漏 → **评测虚高、上线崩塌**。按项目切分保证 test 集项目在训练中完全未见。
- **实现对齐**：切分逻辑由 **C-07 `apps/api/scripts/model3d/dataset_split.py`** 实现（固定随机种子、可一键复现），
  本规范定义口径，C-07 定义脚本与产物（`splits/` 清单 + 数据卡 `DATASHEET.md`）。二者交叉引用，口径必须一致。
- **建议比例**：train : val : test ≈ **7 : 1.5 : 1.5**（按项目数分配，非按图数），每专业批次独立切分保证各域 test 代表性。
- **test 集冻结**：test 集一经切分即**冻结**，训练/调参/选型全程不可触碰，**仅在 C-18 里程碑终评解冻一次**
  （对齐 `docs/PHASE_C_TASKS.md` §4.2 验收总标准与 C-07 验收）。任何在 test 上反复看指标调参 = 变相泄漏，禁止。

## 5. 诚实边界（务必写在评审材料最前，向上管理预期）

1. **类别体系返工代价极高——必须一次评审到位。**
   9 顶层类已被 `layer_conventions.py` / C-04 / C-07 / C-14 全链路锚定；子类体系一旦上线开始标注，
   中途改类别定义会让**已标注数据大面积作废、需重标**。§1 子类表必须经结构/机电/装修专业工程师**联合评审通过**后
   才允许启动大规模标注（C-06）。这是本任务**最高优先级的一次性决策点**。

2. **机电/装修标注专业性强、成本最高，是最大隐形成本。**
   机电管线（风/水/电/消防）与装修长尾符号的判读需要**懂图的专业人员**，非通用众包可替代（对齐 C-06 风险）。
   排期与预算必须为此留足，且**结构先行**——先用成本较低、规范化程度最高的结构域跑通全链路验证方法论，
   再滚动投入机电/装修。

3. **弱标签有噪声，不是终值。**
   C-04 自动标签一致率目标仅 ≥70%——它负责**冷启动规模**，不是金标准。非规范图层/院内私有命名会拉低召回；
   映射表需持续养护（C-04 风险）。训练与评测的**质量底线由 C-06 金标签兜底**。

4. **样本量是量级估算，不是承诺值。**
   §3 门槛为冷启动量级参考，真实需求由 C-14 评测曲线决定——**边采边评、按缺口补量**，
   不承诺"采到 N 张就一定达标"。达标判据永远是 C-09/C-14 的"显著优于零样本 + 纯规则基线"，不是样本计数。

5. **覆盖边界诚实标注。**
   FloorPlanCAD 交叉参照（§1.3）已明示：结构/机电/装修主体类别**开源基线为零**——这既是自建价值，
   也意味着这些域**没有外部参照兜底**，全靠自建数据质量。装修长尾类别可能长期数据不足，
   届时**如实标注"该类别靠规则兜底、学习模型未达标"**，不粉饰（对齐第五章能力边界与 C-18 诚实报告要求）。
