## 0. 三条心智（标注启动前必须建立）

1. **金标签是模型的天花板**。训练/评测集的标注错误会被模型放大，也会污染 C-14 评测基座的可信度。
   宁可标得慢而准，不要快而脏——错误金标签比没有金标签更危险。
2. **弱标签是草稿不是答案**。C-04 的图层/块弱标签只是**预填充**，标注员的职责是**逐个证伪或确认**，
   绝不能默认「机器填的就是对的」而整批放行（否则金标签退化为弱标签，本任务失去意义）。
3. **一致性优先于产量**。验收硬门槛是**双标一致性 Kappa ≥ 0.8**（见 §5）。达不到门槛的批次一律驳回重标，
   不因排期压力放水——数据质量返工的代价远高于当期多标几天。

## 1. 标注类别体系（强制对齐共享 taxonomy）

标注类别**只用** `layer_conventions.py` 固化的 **9 个顶层构件类**，加机电 **4 个系统**维度：

### 1.1 顶层构件类（9 类，`kind`）

| 类别 | code | 说明 | 专业域 |
|---|---|---|---|
| 柱 | `column` | 框架柱、构造柱、剪力墙端柱 | 结构 |
| 梁 | `beam` | 框架梁、次梁、连梁、过梁 | 结构 |
| 板 | `slab` | 楼板、屋面板、悬挑板 | 结构 |
| 墙 | `wall` | 剪力墙、砌体墙、隔墙 | 结构/建筑 |
| 门 | `door` | 各类门（块引用为主） | 建筑 |
| 窗 | `window` | 各类窗（块引用为主） | 建筑 |
| 管 | `pipe` | 给排水/暖通/电气桥架管线 | 机电 |
| 设备 | `equipment` | 水泵/风机/配电箱/末端设备等 | 机电 |
| 轴 | `axis` | 轴网线 + 轴号 | 通用 |

> 类别命名与代码严格一致（`classify_by_layer` 返回值域）。**不得新增顶层类**；无法归入 9 类的符号见 §4 疑难处理。

### 1.2 机电系统维度（4 类，`system`，仅 `pipe`/`equipment` 需标）

`消防` / `给排水` / `电气` / `暖通`（与 `classify_system` 返回值域一致）。
机电构件必须**同时**标 `kind` 与 `system`（例：喷淋管 = `pipe` + `消防`）。

### 1.3 子类体系

- 若 `docs/PHASE_C_DATASET_SPEC.md`（C-05）已定义细化子类（如 `column.构造柱`、`pipe.喷淋`），
  **本规范与之保持一致**，标注按 DATASET_SPEC 的子类粒度执行，顶层类作为其父级归并维度。
- **若 DATASET_SPEC 尚未生成**：以上述 9 顶层类 + 4 系统为准标注，**子类字段留空**并在批次说明中注明
  「子类待 DATASET_SPEC 冻结后回填」。子类体系一经 C-05 评审冻结，历史批次按映射回填，不重标。

## 2. 符号级标注规范

### 2.1 标注对象

标注在 **C-02/C-03 预处理输出的二维图纸**上进行：底图为 SVG（`viewBox` 沿页面点坐标），
每个符号候选是一个**轴对齐外接框（bbox）+ 类别 + 置信度**，溯源字段沿用预处理契约的
`data-id` / `data-layer` / `data-block`（见 `PHASE_C_PREPROCESS_SCHEMA.md` §3）。

### 2.2 四个标注动作（对齐 C-16 交互）

标注员对每个候选框执行且仅执行以下之一，并记录动作类型（用于 C-17 收敛度量）：

| 动作 | 语义 | 触发场景 |
|---|---|---|
| **确认（accept）** | 框与类别都对，采纳为金标签 | 弱标签正确 |
| **否定（reject）** | 该框不是有效构件符号（噪声/误检） | 弱标签虚警 |
| **改类（reclass）** | 框对但类别错，改为正确 `kind`/`system`/子类 | 图层串类、块名误判 |
| **补框（add）** | 机器漏检，人工新增框 + 类别 | 弱标签漏标 |

> 金标签 = 全部 `accept` + `reclass` + `add` 的最终框集合；`reject` 的框不进入金标签但**保留为负样本记录**。

### 2.3 边界（bbox）判定准则

统一准则：**框应紧致包裹符号的全部可视笔画，不多含相邻构件，不遗漏引出线之外的本体**。分类补充：

| 类别 | 边界判定要点 |
|---|---|
| `column` | 包裹柱截面轮廓（含配筋点划线范围）；柱与相接梁/墙**在几何交点处切分**，柱框只含柱本体。 |
| `beam` | 沿梁跨方向包裹梁边线 + 集中标注引出符号的**梁本体**部分；标注文字（如 `KL1(2)`）不计入框，除非该文字块即符号本体。 |
| `slab` | 板为面域，框取板边界闭合区域的外接矩形；仅在有明确板边线/板号时标，纯填充图案不单独成框。 |
| `wall` | 沿墙走向包裹双线墙的两条边线；墙在门窗洞口处**断开为独立段**，各段各一框。 |
| `door` / `window` | 以块引用整体为框（含门扇弧线/窗扇线）；洞口两侧墙不计入。门窗多为 `INSERT` 块，优先按块边界成框。 |
| `pipe` | 沿管线走向的最小外接框；长直管按**图面可见段**成框，遇设备/立管断点切分；须同时标 `system`。 |
| `equipment` | 包裹设备图块本体（不含引出标注文字）；须同时标 `system`。 |
| `axis` | 轴线取贯穿线段外接框，轴号圆圈单独作为同一 `axis` 目标的一部分或按 DATASET_SPEC 子类拆分。 |

**通用容差**：框边与符号笔画的允许间隙 ≤ 图面 2px（按预处理页面点计）；超出即判边界不合格。

### 2.4 疑难情形处理（歧义登记，不得擅自臆断）

| 情形 | 处理 |
|---|---|
| 一图元命中多类（图层/块矛盾） | 以**图面语义**为准改类；无法判定 → 标 `uncertain` 标志，进仲裁。 |
| 符号残缺/被遮挡/跨切图截断 | 标 `truncated` 标志；截断符号按可见部分成框并注明，跨大幅面切片的接边由 C-16 切片策略保证不重复计。 |
| 图层规范外的项目自定义符号 | 归入语义最接近的 9 顶层类 + `custom_layer` 标志，登记原图层名供 C-04 映射表养护。 |
| 无法归入 9 类的图例/说明/图框 | 标 `non_target`（背景），**不进金标签正样本**，但保留供负样本挖掘。 |
| 密集重叠符号（如管线井、配筋密集区） | 逐个成框，允许 bbox 重叠；重叠 IoU 不作为「同一目标」依据，以符号本体计数。 |

> 所有带标志（`uncertain`/`truncated`/`custom_layer`）的样本**强制进入交叉标注与仲裁**，不得单人放行。

## 3. 标注工具选型（复用优先，避免另起炉灶）

### 3.1 首选：复用/扩展前端审校工作台（C-16 深化）

**决策**：标注工具**复用**现有前端 `apps/web/src/pages/model/ProjectModel/DrawingAnnotationQueue.tsx`，

**一套工具两用**（核心收益）：

- **生产审校**：机器出初模 + 置信度，人审确认/修正 → 服务三审文化的成果收敛（C-15/C-16）。

一套工具两用避免了「再起一个独立标注平台」的重复建设、双份数据 schema、双份权限/审计成本，
也让**人审即产数据**（数据飞轮，见 §6）天然成立。

## 4. 质检流程（双人交叉 + 第三方仲裁）

```
弱标签批次(C-04)
   │
   ├─► 标注员 A 独立精标 ─┐
   │                      ├─► 一致性度量(IoU 框级 + Kappa 类级) ──► Kappa ≥ 0.8 ？
   ├─► 标注员 B 独立精标 ─┘                                          │
   │      (A、B 互不可见对方标注)                          ┌── 是 ──┴── 否 ──┐
   │                                                      │                  │
   │                                              分歧样本进仲裁      整批驳回·重标
   │                                                      │
   │                                          第三方仲裁员裁定 ─► 金标签定稿
   │                                                      │
   └──────────────────────────────────► 质检报告(QC_TEMPLATE) + 金标签回流 C-07
```

### 4.1 双人交叉标注

- 每批样本由**两名标注员独立**完成，**互不可见**对方结果（避免锚定偏差）。
- 机电域批次的标注员须为**懂图人员**（能读机电系统图），不可由通用众包替代（见 §7）。
- A/B 完成后系统自动比对，产出一致性度量（§5）与**分歧样本清单**。

### 4.2 第三方仲裁

- 仲裁员为**资深工程师/图审专家**，独立于 A、B。
- 仲裁仅针对**分歧样本 + 全部带标志样本**（§2.4），逐条给出裁定类别/边界与**裁定依据**。
- 仲裁结论为该样本金标签的**最终真值**，写入质检报告可追溯字段。

### 4.3 批次留痕

每批金标签**必须**产出一份 `PHASE_C_ANNOTATION_QC_TEMPLATE.md` 填充实例，含批次号、样本量、
A/B 一致性 IoU/Kappa 数值、分歧样本清单、仲裁结论、通过/驳回判定。**无质检报告的批次不得入库 C-07**。

## 5. 标注一致性度量

用两个正交指标，分别度量**框的定位一致**与**类别的判定一致**：

### 5.1 IoU（框级定位一致性）

- 对 A、B 的框做匹配：两框 **IoU ≥ 0.5** 视为「同一目标」（匹配对）。
- 未匹配框计入**漏配/多配**（一方有、另一方无），反映漏标/虚标分歧。
- 报告**匹配对平均 IoU**（定位紧致度）与**框匹配率**（= 匹配对数 / A、B 并集目标数）。
- 建议参考线：匹配对平均 IoU ≥ 0.7，框匹配率 ≥ 0.85（非硬门槛，供批次健康度观察）。

### 5.2 Cohen's Kappa（类级判定一致性）

- 仅在 **IoU 匹配对**上计算：把每对的「A 类别 vs B 类别」构成混淆矩阵，算 Cohen's Kappa κ。

  ```
  κ = (p_o − p_e) / (1 − p_e)
      p_o = 观测一致比例（对角线之和 / 匹配对总数）
      p_e = 偶然一致比例（按各类别边际概率乘积求和）
  ```

- **机电域**须**分别**报 `kind` 的 κ 与 `system` 的 κ（系统判定错误同样致命）。
- 建议分专业分类别也各报一次 κ，定位长尾类（如装修符号）的低一致来源。

### 5.3 验收门槛（硬约束）

| 指标 | 门槛 | 不达标处理 |
|---|---|---|
| **Cohen's Kappa（类级）** | **κ ≥ 0.8** | **整批驳回重标**（可先复盘规范歧义再重标） |
| 框匹配率（定位） | ≥ 0.85（观察线） | 低于线须在报告分析漏/虚标成因 |

> κ ≥ 0.8 为「几乎完全一致（almost perfect）」区间，是本任务与 `PHASE_C_TASKS.md` C-06 验收标准的一致门槛。
> κ 未达标反映的往往是**规范歧义**而非标注员能力——应回到 §2 收敛判定准则再重标，而非放宽门槛。

## 6. 数据闭环（数据飞轮，对齐 C-15/C-17）

金标签不是终点，而是持续飞轮的一环：

1. **金标签回流 C-07**：定稿金标签 + 质检报告 → C-07 数据集版本管理，进入 train/val/test 切分
   （按项目切分防泄漏），供 C-09 微调与 C-14 评测。
2. **人审修正回流训练标签**：C-15（语义审校）/ C-16（符号审校）**生产环境的人审动作**
   （确认/否定/改类/补框）本身即高价值标注，经同样的质检门槛（§5）后回流为增量训练标签——
   **人审即产数据**，模型越用越准。
3. **收敛度量对齐 C-17**：动作类型（accept/reject/reclass/add）埋点喂 C-17 看板，量化返工点随迭代下降，
   为「AI 出初模 + 人工审改效率提升 25–30%」提供可度量证据。
4. **弱标签映射养护**：改类/自定义图层的登记回流 C-04 `layer_class_map.yaml`，降低下批弱标签噪声，
   形成「弱标签更准 → 人审更省 → 金标签更多」的正循环。

## 7. 诚实边界（写在最后，用于向上管理预期）

1. **标注人力是持续成本，不是一次性投入**。数据集冷启动（C-05）与金标签生产（C-06）是 Phase C 的
   **关键路径与最大隐形成本**，随专业域扩展（结构→机电→装修）滚动投入，不会「标一批就完事」。
2. **机电构件必须懂图人员标注**。管线系统、设备类型、给排水/消防/暖通/电气的区分需要专业读图能力，
   **非通用众包可替代**；错标的机电金标签会让 C-09 机电域模型直接学歪。
3. **金标签质量决定模型上限**。κ ≥ 0.8 门槛不是形式主义——它是防止「脏数据训练脏模型、脏模型污染评测」
   的第一道也是最关键一道闸。
4. **子类体系一旦返工代价极高**。子类以 C-05 `DATASET_SPEC` 一次评审冻结为准，本规范不擅自扩展子类，
   避免历史批次大规模重标。
