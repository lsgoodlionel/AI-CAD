## 1. 这是什么:一分钟看懂

「工程模型」把一个项目里已上传的图纸,**自动升级成一个可在浏览器里旋转查看的三维模型**:楼层按标高一层层堆叠,结构柱、墙、梁、板、机电管线和设备按图纸真实位置重建出来,AI 审图发现的问题、跨图冲突也会以彩色标记球挂在对应楼层上。

你可以用它来:

- **整体感知项目**:一眼看清有几栋楼、多少层、哪层有多少问题。
- **定位问题**:点选三维里的红色标记球,直接跳到出问题的图纸。
- **核对构件**:查看柱/墙/梁/板的位置、数量、截面。
- **参与审校**:对机器识别不确定的地方做「确认/否定/改类/补框」,你的每一次判断都会让模型越来越准。
- **看工程量**:查看混凝土、模板、钢筋的算量汇总。

> **重要认知**:这是「AI 出初模 + 人工在平台内审改」的工作方式,**不是「一键出成品 BIM」**。机器负责把 80% 的确定性活儿干完,不确定的部分交给你快速核对。整体效率提升现实区间约 **25%–30%**。

从 2026-07-14 起,登录后会**默认先进入「项目工作台」**(一个项目一屏总览全流程),而不是直接进模型页——工程模型是流程中的一环。工作台本身的用法见 [§16](#16-项目工作台手册);本手册前 15 章仍聚焦在工程模型工作台本身的详细操作。

## 2. 术语速览

| 术语 | 通俗解释 |
|---|---|
| **构件** | 柱、墙、梁、板、管线、设备这些建筑组成部分 |
| **单体** | 一个项目里的一栋楼(如 1# 楼、裙房) |
| **楼层归属** | 一张图纸属于哪栋楼的哪一层 |
| **标记球** | 三维里悬浮的彩色小球,代表一个审图问题或跨图发现,颜色越红越严重 |
| **LOD** | 模型精细度等级。审图骨架(粗)→ 建筑体量(中)→ 实景近似(细) |
| **渲染模式** | 三维看的方式:IFC 模型 / 构件 / 贴图 / 混合 |
| **符号标注** | 对图纸上的门、窗、柱等图元画框、定类别 |
| **置信度** | 机器对自己识别结果有多大把握,0~1,越低越需要你把关 |
| **返工点** | 你在审校里做了「改类/否定/补框」= 机器初模需要人工修正的地方 |
| **项目工作台(Hub)** | 登录默认页,一个项目一屏看「图纸→审查→建模→算量→创效」走到哪一步了 |
| **审查中心** | 把「单图 AI 审图/跨图/会审/语义/符号」五类问题合并成一个统一清单的页面 |
| **算量中心** | 把「模型 IFC-QTO 算量」与「钢筋翻样」合并展示的统一算量页面 |
| **Finding(问题/发现)** | 审查中心里对五类问题的统一叫法,每条都有「待处理→已确认→已整改→已闭环」状态 |
| **建议待办** | 系统后台在特定条件下生成的「建议重建模型/建议创建创效提案」提示,**只建议,不自动执行**(见 [§16.6](#16-6-自动化建议诚实说明当前状态)) |

## 3. 快速上手(5 分钟跑通)

1. 登录后自动进入**项目工作台**(`/hub`),在项目选择器里选(或搜索)一个项目。
2. 顶部常驻的**流程 Steps**(上传图纸→AI审查→工程建模→算量→创效提案)会高亮你当前所在的一步,点当前步骤下方「去完成 →」直达对应页面;或直接点下方 5 张流水线卡片中的**「工程模型」**卡进入模型工作台。
3. 若提示「该项目尚未生成工程模型」,点 **「立即生成模型」**,页面进入「构建中」并每 5 秒自动刷新进度条。等待其变为「已就绪」(取决于图纸数量,通常数分钟)。
4. 模型就绪后,中间出现三维视图:**左键拖拽旋转,滚轮缩放,右键平移**。
5. 点一个红色标记球 → 右侧抽屉弹出问题详情 → 点「查看图纸」直达源图。
6. 顶部模式切换点到**「审校」**,若显示队列有候选项,展开处理一两条,体验「确认(c)/否定(x)/改类(r)」快捷键。

完成 ✅ 你已经会用核心功能了。下面是逐项详解;想先看新增的项目工作台/上传/审查中心/算量中心整体介绍,可直接跳到 [§16](#16-项目工作台手册)。

## 4. 五个入口:从哪里打开工程模型

工程模型对**所有已登录用户开放**,不限角色。共有五个入口,都会打开同一个模型工作台:

| 入口 | 位置 | 说明 |
|---|---|---|
| **项目工作台** | `/hub` 选定项目后,「工程模型」流水线卡片的进入按钮 | 现在的默认路径:登录 → 工作台 → 点卡片进模型 |
| **主菜单** | 左侧菜单「工程模型」 | 先进项目选择器,再点项目卡片 |
| **图纸详情页** | 图纸详情右上「在工程模型中查看」 | 打开模型并**聚焦到当前这张图纸**(相机对准 + 高亮描边) |
| **项目看板** | 项目看板顶部「工程模型」按钮 | 未选项目时按钮禁用 |
| **图纸列表** | 列表页「工程模型」按钮 | 读取筛选栏已选项目;未选会提示「请先选择项目」 |

> ⚠️ **已知变化**:原来「套图审查详情页」里「在模型中查看跨图发现」这个深链入口,随套图审查页并入[审查中心](#16-3-审查中心)已经**不存在**了——审查中心当前只展示问题清单本身,不再提供跳转到模型三维视图的按钮。如需在模型里核对某个跨图问题涉及的楼层/构件,请用上表任一入口打开该项目模型,自行按楼层/专业筛选查看。

## 5. 主界面导览

模型工作台分为**顶部状态条**和**主体区**,主体区的布局**随「浏览 / 审校 / 算量」三种模式切换而不同**(不再是固定的三栏)。

### 5.1 顶部状态条

- **状态徽标**:构建中(蓝色转圈)/ 已就绪(绿色)/ 构建失败(红色)。
- **版本与时间**:`版本 v3`、`构建于 …`。
- **「重建模型」按钮**:重新跑一遍构建(构建中时禁用)。见 [§6](#6-生成与重建模型)。
- **统计标签**:图纸数、问题数、楼层数、`待人工识别 N`、重建模式(构件级/贴图级/混合)、构件总数、各严重度问题数量。
- **浏览 / 审校 / 算量** 三视图模式切换(下方 §5.2 详解)。
- **LOD 模式按钮组**:审图骨架 / 建筑体量 / 实景近似。不可用的等级会置灰,鼠标悬停有原因说明。
- **渲染模式切换**(当有多种可选时出现):IFC 模型 / 构件 / 贴图 / 混合。
- **构建中**会显示进度条;**失败**会显示红色错误提示。

### 5.2 三视图模式(浏览 / 审校 / 算量)

顶部有一个「浏览 / 审校 / 算量」的分段切换按钮,三个模式**共用同一个中间 3D 视图**,只有左右两侧的工作面板不同:

| 模式 | 左侧面板 | 右侧面板 | 适合做什么 |
|---|---|---|---|
| **浏览**(默认) | 结构导航 + 筛选 + 构件图层 + 模型质量([§9](#9-左栏结构导航)、[§13](#13-模型质量与返工度量浏览模式)) | 无 | 整体看模型、按楼层/单体/专业筛选查看 |
| **审校** | 无 | 审校收件箱 + 语义树候选 + 楼层归属 + 楼层标高校正([§10](#10-审校收件箱)–[§12](#12-楼层归属)) | 处理机器识别不确定的地方 |
| **算量** | 无 | QTO 汇总 + 构件高亮 + 打开算量中心([§14](#14-算量模式与算量中心)) | 快速看算量、跳去完整算量页面 |

> 切换模式**不会**重新加载 3D 场景,相机视角保留。

## 6. 生成与重建模型

### 首次生成

项目从未建过模型时,中间显示空态「该项目尚未生成工程模型」,点 **「立即生成模型」**。

### 重建

图纸有更新、做了人工标注、或想应用最新算法时,点顶部 **「重建模型」**。

- 触发后状态变「构建中」,进度条每 5 秒自动刷新,阶段依次为:抓取输入 → 渲染图纸 → 识别构件 → 组装模型。
- 完成后版本号 +1(如 v3 → v4),旧版本自动被替换。
- **构建是后台异步进行的,你可以离开页面**,稍后回来查看结果。

> 💡 你做的每一次楼层标注、符号确认,都会在下次重建时被吸收进模型。**「标注 → 重建 → 更准」是推荐的迭代节奏。**

## 7. 三维视图操作

### 7.1 基本视角操作(所有渲染模式通用)

| 操作 | 方式 |
|---|---|
| 旋转 | 左键拖拽 |
| 缩放 | 鼠标滚轮 |
| 平移 | 右键拖拽 |
| 选中构件/标记 | 单击(系统会区分单击和拖拽,拖拽超过约 6 像素不算点选) |

视角带阻尼效果,松手后会平滑停下。

### 7.2 四种渲染模式

顶部「渲染模式」切换(具体可选项取决于该模型生成了哪些数据):

- **IFC 模型**:高性能 Fragments 渲染,面向大体量模型,可点选构件查看 IFC 属性。
- **构件**:显示重建出的柱/墙/梁/板等挤出实体。
- **贴图**:把每张图纸作为面板贴在对应楼层。
- **混合**:构件 + 贴图叠加显示。

切换渲染模式时,**相机视角会保留**,不会跳回原点。

### 7.3 LOD 精细度

顶部「LOD 模式」按钮:**审图骨架**(最粗,看结构关系)、**建筑体量**(中等)、**实景近似**(最细)。等级越高需要模型数据越完整,不满足条件的等级会置灰并给出原因。

### 7.4 IFC 属性面板(IFC 模型模式下)

点选构件后,右上角浮出属性面板:IFC 类型、GUID、名称,以及分组的 Pset 参数。点 **「语义树定位」** 可在左栏语义树中反查该构件位置(浏览模式下)。

### 7.5 只看模型本体

三维里除了从图纸识别出的**真实构件**(柱/墙/梁/楼板/管线/设备),还叠加了若干**参照辅助**:一层层半透明的蓝色「楼层板片」(示意楼层堆叠)、图纸贴图/图框面板、体量近似外壳、审图标记球。三维视图右上角有 **「只看模型本体 / 显示全部」** 切换按钮:

- 想更清楚地看真实构件时,点 **「只看模型本体」**——楼层板片、图纸图框面板、外壳、标记球等参照辅助一并隐藏,只保留从图纸识别出的构件本体。
- 再点 **「显示全部」** 恢复。该开关**只影响显示,不改变模型数据**,在浏览/审校/算量各模式下都可用。

## 8. 点选与详情抽屉

在三维里单击对象,右侧弹出详情抽屉,分三种:

- **图纸信息**:图号、图名、专业、阶段、问题数 → 按钮「进入图纸详情」跳转 `/drawings/{id}`。
- **问题标记**:严重度、类型、楼层、专业代码、所属图纸 → 按钮「查看图纸」。
- **构件信息**:构件类型、楼层、数量、标注,并提示「构件级重建(矢量图纸提取)」→ 按钮「查看来源图纸」。

三种模式下都可以点选,不限于浏览模式。

## 9. 左栏结构导航

> 本节内容属于**浏览模式**。

切到**浏览模式**后,左栏是收窄你在三维里看到的内容的面板,**都是即时生效的显示开关,不改变模型数据**。分四张卡片:

### 9.1 结构导航

三个 Tab,点击即可在 3D 视图里「隔离」对应范围(只高亮它,其余变半透明;再点一次取消隔离):

- **语义树**:按单体/分区/功能空间层级浏览与定位。
- **单体**:点击隔离某栋楼;标签标注「人工/识别/无几何」。
- **楼层**:按楼层高低倒序排列,点击隔离。

### 9.2 筛选

- **专业**:勾选结构/建筑/机电/装修,过滤构件与标记。
- **严重度**:勾选严重/较大/一般/提示,过滤问题标记球。
- **标记类型**:图内问题 / 跨图发现。

### 9.3 构件图层(构件/混合渲染模式下)

柱/墙/梁/板/管线/设备/外观壳体的显隐勾选。

### 9.4 模型质量(折叠面板)

默认折叠,展开后即 [§13](#13-模型质量与返工度量浏览模式) 的内容,快速判断当前模型是否可放心用于审图/算量。

## 10. 审校收件箱

> 本节内容属于**审校模式**。

切到**审校模式**,右栏第一个面板(默认展开)是「审校收件箱」——**Phase D 把原来两个分开的队列(符号标注 + 成果审校)合并成了一个统一列表**,一次只处理一件事,不用在两个面板间来回切。

- 列表**同时包含**两类候选:
  - **符号级候选**:门/窗/柱等图元的识别框,颜色 = 置信度(🔴 红 < 50%、🟡 黄 < 75%、🟢 绿较可靠)。
  - **成果审校候选**:拓扑闭合/构件命名/规范符合性等语义正确性判断,冲突项左侧有红色边条并标「规则-模型冲突/未闭合」。
- **排序**:冲突优先,同等条件下低置信度优先。
- **三个动作**:**确认**(✓,识别/判断正确)、**否定**(✗,识别/判断错误)、**改类**(⇄,弹窗选新类别)。
- **补框**(仅符号候选):上方下拉选一张图纸,图上叠加候选框预览,若机器漏识别,选类别 + 框坐标 →「新增框」。
- **键盘快捷键**(流水作业,鼠标不用来回点):`↑/↓` 或 `j/k` 移动焦点、`c` 确认、`x` 否定、`r` 改类。焦点在输入框内时快捷键不生效,避免误触。

> ✅ 你的每一次「确认 / 改类 / 补框」结果都会成为**金标签**,持续训练识别模型——**审校即产数据**,这两条队列共用的返工埋点在 12.2 返工收敛度量 有统一统计。

## 11. 语义树候选

> 本节内容属于**审校模式**。

审校模式右栏第二个面板,面向**单体/分区/功能空间/施工分区**这类结构性语义候选,动作语义和 §10 不同(涉及树结构调整),所以保留为独立面板:

- 每条显示类型(单体/分区/功能空间/施工分区)、状态、置信度。
- 七个操作:**查看证据**、**确认**、**调整父级**、**重命名**、**合并**、**拆分**、**拒绝**。
- 做「重命名/合并/调整父级/拆分」时,弹窗会**实时预览影响范围**(会影响哪些节点/图纸/楼层),确认无误再提交。
- 若期间别人也改了语义树,提交会提示「语义树版本已更新」,点「刷新到最新版」后重做即可。

## 12. 楼层归属

> 本节内容属于**审校模式**。

审校模式右栏第三个面板,处理机器无法自动判断「这张图属于哪层」的图纸,是表单式录入(不是候选确认/否定),所以也是独立面板:

1. 每条显示缩略图/文本线索、图号、置信度。
2. 填三个下拉(支持自动补全):**单体**(可新建/重命名)、**楼层**、**图纸类型**(平面图/立面图/剖面图/节点详图/机电平面)。
3. 三项都填好后点 **「保存标注」**,该条从队列移除。

标注结果会回流用于下次模型重建。

> 审校模式右栏还有第四个面板**「楼层标高校正」**:当 AI 自动打底的楼层标高/层高不准时,可在这里人工录入正确值,修正会累加抬升上层楼层;保存后需点顶部「重建模型」才会生效(该功能沿用「fix/model-3d-quality」分支已交付的标高人工录入通道,本次未变更,仅补充记录入口位置)。

## 13. 模型质量与返工度量(浏览模式)

浏览模式左栏「模型质量」折叠面板,分两部分(子面板小节编号沿用产品早期命名,不代表所属大章节序号):

### 12.1 模型质量

六个指标一览:未分层、楼层冲突、低置信度单体、待人工识别、待审语义、语义冲突。楼层冲突会用提示条列出前几条。

选中某个构件/范围时,还会显示 **LOD 质量**:当前 LOD 等级、已通过的门槛(绿标)、缺失的证据(橙标)、以及为什么降级的说明。

### 12.2 返工收敛度量

- 可按专业筛选(结构/机电/装修/建筑)。
- 四个比率卡片:**确认率**(绿,越高越好)/ **改类率**(黄)/ **否定率**(红)/ **补框率**(蓝)。
- 各专业**返工率进度条**(≥50% 红、≥25% 黄、其余绿)、各类别明细表、按天的返工率收敛趋势图。
- **口径**:返工点 = 改类 + 否定 + 补框;确认 = 机器初模一次通过。趋势下降说明模型在变准。这里的数字对应「25%–30% 效率提升」的现实区间。

## 14. 算量模式与算量中心

### 14.1 模型页内:算量模式

顶部切到**算量模式**,右栏展示:

- **QTO 汇总卡**:混凝土净体积、模板接触面积、钢筋量(有配筋数据时)、覆盖率(已算量/总构件数);按类型分表,点「高亮」可在 3D 里隔离查看该类构件;下方有分楼层小计。
- **构件高亮卡**(构件级模型):勾选类别在 3D 视图中显示/隐藏对应构件,和上面表格的「高亮」联动。
- 卡片右上角**「打开算量中心」**按钮,一键跳到完整的算量中心页面。

### 14.2 算量中心(完整页面)

算量中心是 Phase D 新增的独立页面(`/quantities` 或从项目工作台「算量」卡进入),口径与模型页算量模式**完全一致**(同一份汇总数据),额外提供:

- 分楼层 / 分单体的完整下钻表格。
- **导出 CSV**。
- **钢筋翻样明细下钻**:选择本项目已计算过翻样的图纸,只读查看该图的钢筋总量、优化节约、节约金额、下料废料率;完整的翻样录入/重算入口仍在对应「图纸详情」页(未合并,只是在这里能直接看结果)。

详见 [§16.4](#16-4-算量中心)。

> ⚠️ **关于创效提案的诚实说明**:算量口径本身不会区分「哪个入口更权威」,混凝土/模板/钢筋均为**净体积/接触面积/工程辅助口径**,变截面、地区细则等特殊情况按通用口径处理,正式结算仍需人工复核。若算出钢筋优化有正节约,**当前版本没有「模型页一键生成创效提案」的按钮**——后台虽然预留了对应接口,但尚未接入前端入口;如需据此发起创效提案,请到「创效激励」模块按正常流程手动新建提案(引用测算结果),经济师签字等硬约束照常执行,不会被绕过。审查中心里符合创效潜力判定的问题(Finding)则**已经**有「转创效提案」按钮,见 [§16.3](#16-3-审查中心)。

## 15. 常见问题 FAQ

**Q:点了「立即生成模型」很久还在转圈?**
A:构建是后台异步的,时间取决于图纸数量与类型。可离开页面稍后再来;进度条每 5 秒刷新。若长时间卡住或显示「构建失败」,联系管理员查看后台日志。

**Q:三维里有些图纸是白框/线框,没有贴图?**
A:该图渲染失败或过大时会用线框占位,属正常降级,不影响其余部分。

**Q:为什么某层是空的、构件很少?**
A:可能该层图纸是扫描件、非矢量,或未正确归属楼层。可切到「审校」模式,在「楼层归属」([§12](#12-楼层归属))补标注后重建。

**Q:改了标注,模型没变?**
A:标注保存后需点顶部「重建模型」才会应用到三维。

**Q:构件位置/截面不准?**
A:结构柱/墙/板较可靠;梁为近似(截面可能取默认值);机电为体块示意。当图纸图层命名规范时,地下室**基础底板/筏板/承台**与**地下室外墙**也能被识别(底板/筏板会给更厚的默认厚度)。有剖面图时系统会恢复真实层高与截面,精度更高。请以图纸为准,在审校里修正。

**Q:我没有管理员权限,能用这些功能吗?**
A:能。工程模型的查看、生成、标注、审校对所有登录用户开放,不需要特殊角色。

**Q:以前的「待人工识别」「语义审查」两个面板去哪了?**
A:切到顶部**「审校」模式**——原来的「符号标注」并进了新的「审校收件箱」([§10](#10-审校收件箱)),原来的「成果审校」也并进了同一个收件箱;「语义候选队列」([§11](#11-语义树候选))和「楼层归属」([§12](#12-楼层归属))保留为独立面板,动作语义没变。

**Q:上传图纸的入口怎么只剩一个了?以前分单张/批量/整套三个模式的入口呢?**
A:图纸列表页的上传弹窗现在是**统一上传向导**:拖入文件后系统自动判断是单张、批量还是 ZIP 整套,不需要你先选模式,详见 [§16.2](#16-2-统一上传向导)。

**Q:「套图审查」页面打不开了/自动跳到别的地方?**
A:套图审查已并入**审查中心**(`/review`),旧链接/收藏夹里的 `/drawings/review-batches` 会自动跳转过去,不是故障,详见 [§16.3](#16-3-审查中心)。

## 16. 项目工作台手册

> 本章覆盖 Phase D 新增/合并的平台级功能:登录默认页、统一上传、审查中心、算量中心、就地帮助气泡,以及后台的「自动化建议」编排层。这些内容围绕工程模型展开,但不局限于模型工作台本身,故单列一章。

### 16.1 项目工作台(Hub)

- **入口**:登录后自动进入 `/hub`;也可随时点左侧菜单「项目工作台」回到这里。带项目 ID 访问 `/projects/{id}/hub` 直达指定项目。
- 未选定项目时,展示**项目选择器**;选定项目后展示工作台主体,包含三部分:
  1. **顶部流程 Steps**:「上传图纸 → AI审查 → 工程建模 → 算量 → 创效提案」五步,当前所在步骤高亮,并在该步下方显示「去完成 →」按钮,点击直达对应模块(建模/算量两步都指向工程模型页,创效指向创效激励模块)。
  2. **5 张流水线卡片**:图纸(按状态统计)、审查(AI 质量概况)、建模(项目模型状态,点击进入模型工作台)、算量(点击进入算量中心)、创效(提案漏斗 + 年度节约额 + KPI 红线预警)。响应式布局:大屏 3 列、平板 2 列、手机 1 列。
  3. **近期活动**:最近发生的关键动作(上传图纸/一审通过/经济师签字/发布图纸/提交提案/审批提案)时间线。
- 你上次打开的项目会被记住(本地存储),下次进工作台时更方便定位。

### 16.2 统一上传向导

图纸列表页的上传入口重构为**一个拖拽弹窗**,不用你先选「单张 / 批量 / ZIP 整套」哪种模式:

- 拖入或选择文件后,系统按你选中的文件**自动判定**上传类型:
  - 1 个非压缩文件 → **单张上传**,弹窗内可核对/编辑图号、专业、版本、标题,可填预估影响金额。
  - 多个非压缩文件 → **批量上传**,表格逐条核对元数据(图号按文件名自动预解析,专业按「结施/建施/水施/电施/暖施/装施」等关键词预判)。
  - 1 个 `.zip` → **ZIP 整套导入**,上传后自动解压并按文件名解析每张图纸信息,无需逐张填写。
  - 同时选了 ZIP 和普通文件,或选了多个 ZIP,会提示「不能同时选择/一次只能导入一个」,需要分开操作。
- 支持 PDF / DWG / DXF / IFC,单文件 ≤200MB。
- 底部有一个勾选框**「上传完成后自动创建套图审查批次」**:勾选后,上传成功的图纸会被自动加入一个新的套图审查批次,上传完直接跳到审查中心查看该批次的跨图分析结果,不用回图纸列表再手动建批次。
- 上传成功后会自动触发对应的 AI 审图任务(单张审图/整套多任务并发),你不需要额外点击。

### 16.3 审查中心

- **入口**:左侧菜单「审查中心」(`/review`),或 `/projects/{id}/review` 直达某项目;旧的「套图审查」独立页面路由(`/drawings/review-batches`、`/drawings/review-batches/:id`)已**自动重定向**到这里,书签/历史链接不会失效。
- 把原来分散的**单图 AI 审图 / 会审审查 / 套图跨图审查**,再加上**语义审校**、**符号待审**,合并成一份统一问题清单(内部统称 **Finding**),按来源分 6 个 Tab:**全部 / 单图 / 跨图 / 会审 / 语义 / 符号**。
- **闭环状态机**:每条 Finding 有 `待处理 → 已确认 → 已整改 → 已闭环` 四态,**只能顺向流转、不可回退**;点「确认」「标记已整改」「闭环」按钮逐步推进,可附一句备注。若因为并发被别人改过状态,系统会返回冲突提示,刷新后重试即可。
- **转创效提案**:当某条 Finding 被规则(必要时辅以 LLM)判定为**有创效潜力**时,详情区会出现「转创效提案」按钮,点击即创建一条创效提案**草稿**并跳转到提案详情页;没有创效潜力的问题不显示该按钮,点击后台判定不通过也会有友好提示,不会莫名其妙报错。这一步和上面的状态流转互不影响——哪怕问题已闭环,只要有创效潜力仍可以补建创效线索。**草稿之后仍要走正常的三审创效签字流程**,不会绕过经济师签字等硬约束。
- 审查中心目前**不提供**跳转到工程模型三维视图的按钮(见 [§4](#4-五个入口从哪里打开工程模型) 的已知变化说明);需要在模型里核对某条问题,请自行从模型的入口打开对应项目。

### 16.4 算量中心

- **入口**:左侧菜单「算量中心」(`/quantities`),或从项目工作台「算量」卡片、模型页「算量模式」的「打开算量中心」按钮进入。
- 把原来分散在**模型页(IFC-QTO 算量)**与**图纸详情(钢筋翻样)**两处、口径不完全一致的算量入口,统一到一个页面:顶部同一口径展示混凝土净体积/模板接触面积/钢筋量,下方分楼层/分单体下钻表格,支持导出 CSV,再往下是钢筋翻样明细下钻区(选图纸只读查看翻样结果)。
- 钢筋翻样的**完整录入/重算**依然在对应图纸详情页完成(算量中心的翻样区是只读展示 + 一键跳转,不是替代录入入口)。
- 模型页本身仍保留一份「算量模式」面板(见 [§14.1](#14-1-模型页内算量模式)),作为「模型里顺手看一眼 + 构件高亮联动」的轻量视图,和算量中心数据来源相同,不会出现两处对不上的情况。

### 16.5 就地帮助气泡

- 多处专业面板(审查中心统计条/筛选栏、模型质量面板、审校收件箱、语义树候选、楼层归属、模型左栏「结构导航」等)的标题旁,会有一个小号「？」图标。
- 鼠标悬停或点击弹出 1~2 句简短中文解释;部分气泡底部有**「查看手册 →」**链接,点击会跳转到 `/help` 帮助中心并**自动定位到本手册对应小节**(深链锚点按小节标题生成,不需要你手动翻章节)。
- 这是「就地」的快速提示,不会取代完整手册——遇到复杂问题仍建议来 `/help` 或本手册查详细说明。

### 16.6 自动化建议(诚实说明,当前状态)

平台后台有一个轻量的「事件编排层」,会在特定条件下自动生成两类**建议待办**:

1. **审图完成 → 建议重建模型**:某项目的 AI 审图任务完成、且自上次建模以来变更的图纸数超过阈值时生成。
2. **模型建成 → 建议创建创效提案**:模型重建完成后,若算出的钢筋等节约额超过阈值时生成。

**请注意以下两点,避免误解为「全自动」**:

- **这只是建议,不会自动执行任何硬动作**——采纳建议本身**不会**自动帮你点「重建模型」或自动生成创效提案,你仍需要在对应页面手动操作。
- **当前版本的前端界面还没有展示这些建议待办的入口**(不在项目工作台、审查中心或模型页的任何角落弹出提醒)。这是一个已知的、暂未完成的产品缺口,不是你操作有误。如果你确实需要查看,请联系管理员通过后台接口查询(见管理员手册)。

因此,现阶段判断「图纸变了要不要重建模型」「算量出来了要不要发提案」,仍请按本手册前面各章描述的常规流程手动操作——项目工作台的流程 Steps 和流水线卡片会如实反映当前状态,可以作为你判断的依据。
